# Astro Static Template

Нейтральный шаблон статического сайта на Astro, React и TypeScript.

Шаблон содержит минимальный пример страницы, подключенный React-компонент, тесты и набор проверок для разработки статических сайтов. Документы о целях, аудитории, страницах и границах создаются уже в новом проекте, созданном из этого template-репозитория.

## Стек

- Astro.
- React.
- TypeScript.
- Vitest.
- ESLint.
- Prettier.
- Stylelint.
- PostCSS.

## Быстрый старт

Используй Node.js 24.

```bash
make install
make dev
```

После запуска Astro покажет локальный адрес разработки в терминале.

## Команды разработки

```bash
make help
make install
make dev
make typecheck
make format
make format-check
make lint
make lint-fix
make test
make build
make check
```

Основной интерфейс разработки находится в `Makefile`. `package.json` хранит атомарные команды инструментов, которые вызывает `Makefile`.

## Структура проекта

- `src/pages/index.astro` - стартовая страница шаблона.
- `src/lib/site.ts` - нейтральные данные и функции примера.
- `src/lib/site.test.ts` - тесты примера.
- `src/components/InfrastructureCheck.tsx` - React-компонент для проверки инфраструктуры.
- `src/styles/global.css` - глобальные стили.
- `docs/` - легкая агентская система, шаблоны задач, спецификаций, ADR, ревью и инженерные правила.
- `agents/` - роли агентов для постановки, проектирования, реализации, ревью, закрытия и сервисного режима.
- `AGENTS.md` - базовые правила работы агентов.

## Агентская система

В template оставлена облегченная нейтральная агентская система:

- мелкие правки можно делать без `TASK`, `SPEC`, `ADR` и `REVIEW`;
- `TASK` обязателен только для значимых изменений;
- `ADR` нужен только при выборе архитектуры, стека, зависимости, хранения данных, деплоя или интеграции;
- `SPEC` нужна для значимых изменений с несколькими файлами, новым поведением или проверяемыми критериями;
- `REVIEW` нужен перед закрытием значимого изменения;
- `RUN` журналы не обязательны для каждой задачи и используются для ретроспективы или режима `РЕМОНТ`.

Постоянные правила находятся в `AGENTS.md`, повторяемый порядок работы - в `docs/agent-workflow.md`.

## Git hooks

После `npm install` пакет `simple-git-hooks` устанавливает локальный `pre-push` hook.

Перед `git push` автоматически запускается:

```bash
make lint
```

Для ручной полной проверки используй:

```bash
make check
```

## Стили

Глобальные стили находятся в `src/styles/global.css`.

Проект использует легкий PostCSS-слой без Sass и utility-first фреймворков:

- `postcss-nesting` - вложенные CSS-селекторы.
- `postcss-custom-media` - именованные breakpoint через `@custom-media`.
- `autoprefixer` - браузерные префиксы по Browserslist.

## Проверка документации

Для документационных изменений:

1. Прочитай измененные файлы.
2. Проверь отсутствие противоречий с `AGENTS.md`.
3. Проверь русский язык и терминологию.
4. Укажи в итоговом сообщении, если автоматическая проверка отсутствует.
