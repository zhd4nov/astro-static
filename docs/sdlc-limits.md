# Лимиты SDLC

## Назначение

Этот документ задает предохранители от бесконечных циклов между агентами.

Лимит останавливает повтор одного и того же этапа без новой информации.

## Общие правила

1. Считай проходы внутри одной задачи.
2. Фиксируй каждый проход в истории статусов задачи.
3. Не начинай новый проход, если лимит исчерпан.
4. При исчерпании лимита поставь задаче статус `Заблокирована`.
5. Укажи причину блокировки в задаче.
6. Обнови `docs/tasks/index.md`.
7. Верни решение пользователю или предыдущему ответственному агенту.

## Постановка задачи

Максимум: 1 раунд уточняющих вопросов.

После раунда вопросов:

1. Создай задачу с явными допущениями.
2. Или поставь статус `Предложена` и зафиксируй блокирующие вопросы.

Не задавай повторный раунд вопросов без новой информации от пользователя.

## Проектирование

Максимум: 2 прохода подготовки ADR или спецификации.

После второго прохода без готового решения:

1. Зафиксируй блокирующий вопрос.
2. Поставь задаче статус `Заблокирована`.
3. Передай решение пользователю.

Не переписывай ADR или спецификацию по кругу без нового ограничения или решения.

## Реализация

Максимум: 3 попытки исправить одну категорию ошибки.

Категория ошибки:

1. Проверка типов.
2. Линтер.
3. Форматирование.
4. Тесты.
5. Сборка.
6. Пользовательский сценарий.
7. Замечание ревью.

После третьей неуспешной попытки:

1. Останови реализацию.
2. Зафиксируй причину.
3. Поставь задаче статус `Заблокирована`.
4. Передай задачу пользователю или архитектурному агенту.

## Ревью

Максимум: 3 прохода ревью: `r1`, `r2`, `r3`.

После `r3` без статуса `Принято`:

1. Поставь задаче статус `Заблокирована`.
2. Зафиксируй последнее ревью.
3. Укажи, почему задача не принята.
4. Передай решение пользователю или архитектурному агенту.

Повтор одного блокирующего замечания в двух ревью подряд требует анализа причины.

Новое блокирующее замечание после `r2` требует объяснения, почему оно не было найдено раньше.

## Возврат к архитектуре

Возвращай задачу архитектурному агенту только если проблему нельзя исправить в рамках текущей спецификации.

Причины возврата:

1. Спецификация противоречит ADR.
2. Критерий приемки непроверяемый.
3. Требование технически невыполнимо в выбранных ограничениях.
4. Для исправления нужно изменить стек, архитектуру, зависимости или границы задачи.

Не возвращай задачу архитектурному агенту из-за обычной ошибки реализации.

## Блокировка

Блокировка не закрывает задачу.

Блокировка означает, что следующий проход запрещен без нового решения.

В задаче укажи:

1. Исчерпанный лимит.
2. Последний артефакт.
3. Повторяющуюся причину.
4. Требуемое решение.
