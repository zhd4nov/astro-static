# Лимиты процесса

## Назначение

Этот документ ограничивает повторные проходы, чтобы задача не уходила в бесконечный цикл.

## Лимиты

| Область | Лимит | После лимита |
| --- | --- | --- |
| Уточнения | 1 раунд | Зафиксировать блокирующую неоднозначность и остановиться |
| `ADR` | 2 прохода | Зафиксировать спорное решение и заблокировать задачу |
| `SPEC` | 2 прохода | Зафиксировать нерешенное противоречие и заблокировать задачу |
| Реализация | 3 попытки на одну категорию ошибки | Зафиксировать причину и заблокировать задачу |
| Файловое ревью | `r1`, `r2`, `r3` | После `r3` зафиксировать причину и заблокировать задачу |

## Правило блокировки

После исчерпания лимита:

1. Не продолжай тот же цикл без нового решения пользователя.
2. Зафиксируй причину блокировки в `TASK`, если задача отслеживается.
3. Обнови `docs/tasks/index.md`, если задача отслеживается.
4. Укажи блокировку и невыполненные проверки в итоговом сообщении.
