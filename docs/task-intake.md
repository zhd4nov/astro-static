# Постановка задачи

## Назначение

Этот документ задает короткий порядок обработки запроса перед выбором уровня задачи.

## Порядок

1. Сохрани намерение пользователя без подмены смысла.
2. Отдели цель, границы, ограничения и критерии готовности.
3. Зафиксируй неизвестное.
4. Выбери минимальный уровень по `docs/agent-workflow.md`.
5. Создай `TASK` только для обычной или сложной задачи.

## Вопросы

Задавай вопросы только при блокирующей неоднозначности, когда без ответа нельзя безопасно выбрать цель, границы или проверку.

Если можно продолжить с разумным допущением, зафиксируй допущение и продолжай.

## Результат

После постановки должно быть понятно:

1. Что нужно изменить.
2. Что не входит в задачу.
3. Как проверить готовность.
4. Нужен ли `TASK`.
5. Нужны ли `ADR`, `SPEC`, файловое `REVIEW` или `RUN`.
