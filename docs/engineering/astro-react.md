# Astro и React

## Назначение

Этот документ задает правила для проекта на Astro с React-компонентами.

Базовые правила TypeScript находятся в `docs/engineering/typescript.md`. Не дублируй их здесь.

## Роль Astro

1. Используй Astro как основной слой страниц, маршрутов и статической сборки.
2. Создавай страницы в `src/pages/`.
3. Оставляй информационные страницы статическими, если им не нужна интерактивность.
4. Не добавляй серверный адаптер Astro без отдельного ADR.
5. Не превращай весь сайт в клиентское приложение внутри Astro.

## Роль React

1. Используй React только для компонентов, которым нужна интерактивность или которые проще поддерживать как React-компоненты.
2. Не добавляй директивы гидратации `client:*` без явной пользовательской необходимости.
3. Если React-компонент используется только для статической разметки, оставляй его без клиентской гидратации.
4. Держи состояние внутри минимального интерактивного острова.
5. Не поднимай клиентское состояние на уровень страницы без необходимости.

## Структура

1. Страницы храни в `src/pages/`.
2. Переиспользуемые React-компоненты храни в `src/components/`.
3. Общую типизированную логику храни в `src/lib/`.
4. Не смешивай текст сайта, маршрутизацию и интерактивную логику в одном файле без необходимости.
5. Общие типы экспортируй рядом с владельцем данных.

## Типизация

1. Все экспортируемые функции и компоненты должны иметь понятный публичный контракт.
2. Для свойств React-компонентов используй явные типы `Readonly`.
3. Используй `import type` для импортов типов.
4. Не используй `React.FC` как способ скрыть контракт компонента.
5. Не обходи типизацию через `any`, `as`, non-null assertion или директивы подавления ошибок.

## Проверки

1. Проверяй Astro и TypeScript командой `npm run typecheck`.
2. Проверяй форматирование командой `npm run format:check`.
3. Проверяй правила кода командой `npm run lint`.
4. Проверяй тесты командой `npm run test`.
5. Проверяй статическую сборку командой `npm run build`.
6. Перед ревью запускай `npm run check`.

## Тестирование

1. Покрывай тестами общую логику из `src/lib/`.
2. Добавляй тесты React-компонентов, когда в них появляется условная логика или пользовательское состояние.
3. Проверяй маршруты и рендеринг после появления реальных страниц.
4. Не считай прохождение Vitest заменой `astro check` и сборке.
