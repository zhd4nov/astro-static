# Журналы процесса

## Назначение

Эта директория хранит данные о выполнении значимых задач, если журнал нужен для ретроспективы или разбора процесса.

Режим `РЕМОНТ` использует эти данные для улучшения системы.

## Правила

1. Создавай журнал только если он нужен для ретроспективы, сложного изменения, сервисного режима или явно запрошен.
2. Используй шаблон `docs/process-runs/template.md`.
3. Обновляй `docs/process-runs/index.md`.
4. Не храни здесь требования сайта, которые должны жить в проектных документах.
5. Не меняй правила на основании журнала без отдельной задачи.
6. Ссылайся на TASK, SPEC, ADR и REVIEW.
7. В режиме `РЕМОНТ` допускается создавать ретроспективный журнал.
8. Заполняй обязательные разделы по типу журнала.

## Имена файлов

Используй формат:

`RUN-YYYY-MM-DD-short-name.md`

Пример:

`RUN-2026-05-01-first-site-slice.md`
