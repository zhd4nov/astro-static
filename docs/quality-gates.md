# Проверки качества

## Назначение

Этот документ задает операционный порядок проверки результата.

## Порядок

1. Определи затронутые файлы.
2. Определи доступные команды из `README.md` и `Makefile`.
3. Запусти применимые проверки.
4. Проверь пользовательский сценарий, если он есть.
5. Укажи невыполненные проверки и причину.

## Команды

Для кода используй команды из `README.md`.

Обычно применимы:

- `make format-check`
- `make lint`
- `make test`
- `make build`
- `make check`

Для документационных изменений дополнительно проверь:

1. Отсутствие противоречий с `AGENTS.md`.
2. Отсутствие устаревших ссылок.
3. Русский язык и терминологию.
4. То, что один документ не пересказывает другой.

## Отчет

В итоговом сообщении укажи:

1. Какие проверки выполнены.
2. Какие проверки не выполнены.
3. Почему проверка не выполнена, если такое есть.
4. Что нужно сделать следующим шагом, если проверка выявила блокер.
