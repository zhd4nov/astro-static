# Проверки качества

## Назначение

Этот документ задает обязательные проверки перед завершением задачи.

Конкретные команды добавляй после утверждения стека и спецификации разворачивания проекта.

## Общий порядок

1. Определи затронутые файлы.
2. Определи доступные команды проверки.
3. Запусти применимые проверки.
4. Проверь измененный пользовательский сценарий, если он есть.
5. Зафиксируй невыполненные проверки и причину.
6. Не пропускай обязательную проверку молча.

## Статусы проверки

Используй один статус:

- `выполнена`;
- `не применима`;
- `невозможна из-за среды`;
- `не настроена`;
- `провалена`.

Не принимай задачу, если обязательная проверка имеет статус `провалена`.

Не принимай задачу, если обязательная проверка имеет статус `не настроена` после утверждения стека.

Статусы `не применима` и `невозможна из-за среды` допустимы только с причиной.

## Документация

Для документационных изменений:

1. Прочитай измененные файлы.
2. Проверь отсутствие противоречий с `AGENTS.md`.
3. Проверь отсутствие дублирования правил.
4. Проверь русский язык и терминологию.

## Код

Для изменений кода:

1. Запусти проверку типов.
2. Запусти форматирование, если оно настроено.
3. Запусти линтер, если он настроен.
4. Запусти тесты, если они есть.
5. Запусти сборку, если она есть.
6. Проверь измененный сценарий вручную, если автоматической проверки недостаточно.
7. Если команда проверки типов отсутствует после утверждения стека, укажи статус `не настроена`.

## Интерфейс

Для изменений интерфейса:

1. Запусти проект локально, если это возможно.
2. Проверь основной пользовательский сценарий.
3. Проверь отсутствие видимых поломок верстки.
4. Проверь состояние на узком и широком экране, если изменение влияет на адаптивность.

## Команды после утверждения стека

После утверждения стека и спецификации разворачивания проекта зафиксируй команды в `README.md`.

Минимальный набор:

- установка;
- локальный запуск;
- проверка типов;
- форматирование;
- линтер;
- тесты;
- сборка.

Если используется монорепозиторий, добавь команды из корня:

- проверить весь репозиторий;
- проверить отдельную часть;
- собрать весь репозиторий;
- собрать отдельную часть.
