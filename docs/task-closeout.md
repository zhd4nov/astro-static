# Закрытие задачи

## Назначение

Этот документ задает порядок закрытия отслеживаемой задачи.

## Порядок

1. Сверь результат с целью и критериями готовности.
2. Зафиксируй выполненные проверки.
3. Зафиксируй невыполненные проверки и причину.
4. Обнови итог закрытия в `TASK`.
5. Поставь статус `Принята` или `Заблокирована`.
6. Обнови `docs/tasks/index.md`.

## Ограничения

1. Не добавляй новую работу в закрываемую задачу.
2. Не закрывай задачу, если критерии готовности не проверены и причина не зафиксирована.
3. Не скрывай проваленную или невозможную проверку.
4. Правила системы меняй только в режиме `РЕМОНТ`.
