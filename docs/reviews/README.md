# Результаты ревью

## Назначение

Эта директория хранит результаты ревью значимых изменений.

## Правила

1. Создавай файл ревью для каждого значимого изменения.
2. Используй шаблон `docs/reviews/template.md`.
3. Связывай ревью со спецификацией.
4. Связывай повторное ревью с предыдущим ревью.
5. Не перезаписывай существующий файл ревью.
6. Не храни здесь черновики спецификаций или ADR.
7. Не создавай ревью после `r3` без нового решения пользователя или архитектурного агента.

## Имена файлов

Используй формат:

`REVIEW-NNNN-rX-short-name.md`

Где:

- `NNNN` соответствует задаче или спецификации.
- `rX` обозначает проход ревью: `r1`, `r2`, `r3`.
- `short-name` кратко описывает задачу.

Пример:

`REVIEW-0003-r1-home-specialists-slice.md`

Повторное ревью той же задачи:

`REVIEW-0003-r2-home-specialists-slice.md`

## Версионирование

1. Первый проход ревью всегда получает `r1`.
2. Каждый следующий проход увеличивает номер на 1.
3. Повторное ревью всегда создает новый файл.
4. Предыдущий файл ревью не изменяется.
5. Последнее ревью со статусом `Принято` разрешает закрытие задачи.
6. После `r3` без приемки применяй `docs/sdlc-limits.md`.
