# РЕМОНТ процесса

## Назначение

Этот документ задает правила сервисного режима `РЕМОНТ`.

## Запуск

1. Режим запускается только по явному слову `РЕМОНТ` в запросе пользователя.
2. Без `РЕМОНТ` нельзя менять системные документы.
3. В режиме `РЕМОНТ` не выполняй продуктовые задачи сайта.

## Защищенные системные документы

К защищенным системным документам относятся:

1. `AGENTS.md`.
2. `docs/agent-workflow.md`.
3. `docs/agent-roles.md`.
4. `docs/task-management.md`.
5. `docs/task-intake.md`.
6. `docs/task-closeout.md`.
7. `docs/quality-gates.md`.
8. `docs/review-standards.md`.
9. `docs/sdlc-limits.md`.
10. `docs/process-improvement.md`.
11. Шаблоны и README внутри `docs/tasks/`, `docs/reviews/`, `docs/specs/`, `docs/adr/`, `docs/process-runs/`.

## Когда менять правила

Меняй правила только если есть одно из оснований:

1. Явный запрос пользователя.
2. Противоречие между документами.
3. Повторяющаяся проблема процесса.
4. Блокировка задачи из-за правил.
5. Риск порчи результата.
6. Устаревшая сложность, которая мешает работе.

## Ограничения

1. Не меняй правила системы в продуктовой задаче.
2. Не добавляй правило без понятного основания.
3. Не дублируй одно правило в нескольких документах.
4. Не превращай временное решение в постоянное правило без подтверждения.
5. После изменения правил проверь ссылки и отсутствие противоречий.
