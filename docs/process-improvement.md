# Улучшение процесса

## Назначение

Этот документ задает сервисный режим улучшения агентной системы.

Режим улучшения отделен от рабочих задач сайта.

## Запуск

Запускай режим только по явному слову `РЕМОНТ` в запросе пользователя.

Без слова `РЕМОНТ` не анализируй систему правил и не меняй защищенные системные документы.

## Жесткий запрет

Во время рабочей задачи сайта запрещено менять:

1. `AGENTS.md`.
2. `agents/`.
3. `docs/agent-workflow.md`.
4. `docs/task-intake.md`.
5. `docs/task-management.md`.
6. `docs/task-closeout.md`.
7. `docs/sdlc-limits.md`.
8. `docs/review-standards.md`.
9. `docs/quality-gates.md`.
10. `docs/engineering-standards.md`.
11. `docs/adr/template.md`.
12. `docs/specs/template.md`.
13. `docs/reviews/README.md`.
14. `docs/reviews/template.md`.
15. `docs/tasks/README.md`.
16. `docs/tasks/template.md`.
17. `docs/process-improvement.md`.
18. `docs/process-runs/README.md`.
19. `docs/process-runs/template.md`.

Исключение: пользователь явно запустил режим `РЕМОНТ`.

Журналы `docs/process-runs/RUN-*.md` и реестр `docs/process-runs/index.md` являются данными процесса. Их обновляют при создании журнала процесса.

Документы `docs/engineering/` можно менять в рабочем режиме только по ADR или SPEC выбора стека.

## Данные

Используй только зафиксированные артефакты:

1. TASK.
2. SPEC.
3. ADR.
4. REVIEW.
5. Итоги закрытия.
6. `docs/tasks/index.md`.
7. Журналы из `docs/process-runs/`.

Не используй догадки как основание для изменения правил.

## Что фиксировать

Во время закрытия значимой задачи фиксируй:

1. Task ID.
2. Итоговый статус.
3. Наличие ADR.
4. Наличие SPEC.
5. Количество ревью.
6. Возвраты на доработку.
7. Блокировки.
8. Невыполненные проверки.
9. Повторяющиеся замечания.
10. Кандидаты на улучшение.

В режиме `РЕМОНТ` анализируй накопленные данные и фиксируй:

1. Повторяющиеся проблемы.
2. Причины блокировок.
3. Повторяющиеся возвраты на доработку.
4. Противоречия правил.
5. Кандидаты на изменение системы.

## Решения

Не меняй правила сразу по единичному неудобству.

Создавай задачу на изменение системы, если:

1. Проблема повторилась.
2. Проблема заблокировала задачу.
3. Проблема создала риск порчи результата сайта.
4. Правило противоречит другому правилу.

## Выход

Результат режима `РЕМОНТ`:

1. Анализ журналов процесса.
2. Выводы на основе фактов.
3. Кандидаты на улучшение.
4. Отдельные задачи на изменение системы, если они нужны.

Если создан ретроспективный журнал, обнови `docs/process-runs/index.md`.
