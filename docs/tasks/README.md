# Задачи

## Назначение

Эта директория хранит уточненные задачи, которые передаются в агентную систему.

Задача фиксирует пользовательский запрос в управляемом виде и связывает все последующие артефакты.

## Правила

1. Создавай задачу для каждого значимого изменения.
2. Перед созданием задачи обработай запрос по `docs/task-intake.md`.
3. Используй шаблон `docs/tasks/template.md`.
4. Присваивай задаче `Task ID`.
5. Связывай задачу с ADR, спецификациями и ревью.
6. Обновляй статус задачи при переходе между этапами.
7. Обновляй `docs/tasks/index.md` после каждого изменения статуса или связанных артефактов.
8. Заполняй итог закрытия после принятого ревью.

## Имена файлов

Используй формат:

`TASK-NNNN-short-name.md`

Пример:

`TASK-0001-select-site-stack.md`

## Реестр

Общий реестр задач находится в `docs/tasks/index.md`.

Реестр должен показывать:

- все задачи;
- текущий статус;
- следующий агент;
- связанный ADR;
- связанную спецификацию;
- все ревью;
- последнее ревью.
