# Стандарты ревью

## Назначение

Этот документ задает, когда нужен отдельный файл ревью и как фиксировать результат проверки.

## Формат по умолчанию

По умолчанию ревью фиксируется чеклистом в итоговом сообщении или в `TASK`, если задача отслеживается.

## Когда нужен REVIEW-файл

Создавай `docs/reviews/REVIEW-*.md` только если есть одно из оснований:

1. Архитектурный риск.
2. Возврат на доработку.
3. Спорная приемка.
4. Явный запрос пользователя.

## Статусы

Используй только эти статусы:

- `Принято`
- `Требуются изменения`
- `Заблокировано`

## Блокеры

Блокирующим считается замечание, если изменение:

1. Не выполняет цель задачи.
2. Нарушает критерии готовности.
3. Нарушает `AGENTS.md`.
4. Противоречит принятому `ADR`.
5. Ломает существующий пользовательский сценарий.
6. Добавляет необоснованную сложность или зависимость.
7. Скрывает невыполненную или проваленную проверку.
8. Расширяет скоуп без запроса пользователя.
